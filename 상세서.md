# Sound Mind(가칭) UX/UI 상세서 v1 (Light Only)

- 작성일: 2026-02-04
- 대상: PO / 디자이너 / iOS·Android·Web 개발
- 범위: Mobile App(iOS/Android) + Responsive Web(PC/Mobile)
- 테마: **Light Only(1차 고정)**
- 톤앤매너: **Calm Nature** (따뜻한 뉴트럴 배경 + 세이지/티얼 포인트, 비판단적·안정적 커뮤니케이션)

---

## 0) 목적 & 적용 범위

- 목표: “음성 녹음 → 분석 → 이해 가능한 시각화 → 즉시 실행(명상/가이드)”까지 사용자가 **불안 없이** 따라갈 수 있는 UX/UI 기준을 문서화합니다.
- 본 문서는 **규칙/규격 중심** 문서이며, 화면·컴포넌트·토큰·상태를 개발/디자인이 그대로 참고해 구현할 수 있도록 작성합니다.
- 기술 스택이 확정되지 않았으므로 규격은 **플랫폼-중립(px/dp)** 기준으로 정의합니다.
- 다크모드: 본 버전은 **Light Only**로 고정합니다(다크는 토큰 확장으로 2차).

---

## 1) 제품 경험 목표(UX Principles)

1. **안심(Trust & Safety)**: 의료적 진단이 아닌 “상태 참고”임을 명확히 하고, 데이터/프라이버시를 UI에서 반복 확인합니다.
2. **인지부하 최소(Clarity)**: 녹음/분석/결과의 핵심 CTA는 **화면당 1개**를 기본으로 고정하고, 보조 정보는 접기/카드로 정리합니다.
3. **비판단적 언어(Non-judgmental copy)**: “좋음/나쁨” 대신 “경향/변화/추천” 중심으로 표현합니다.
4. **루틴화(Routine)**: 오늘의 상태 → 짧은 행동(명상 3~10분)으로 자연스럽게 이어지는 플로우를 제공합니다.
5. **연속성(Continuity)**: 단발 점수보다 “추세”를 기본으로 하며, 비교는 “어제/지난주 평균”처럼 부드럽게 제공합니다.

## 2) 정보구조(IA) & 네비게이션

### 2.1 주요 메뉴(권장)

- 탭(모바일): `홈` / `녹음` / `치유` / `보관함`
- 웹: 상단 네비(동일 메뉴) + 우측 `프로필/설정`

### 2.2 1차 화면 맵

- 온보딩: `환영` → `약관/개인정보 동의` → `마이크 권한` → (선택) `첫 녹음 가이드`
- 홈/대시보드: 요약 + 추세 + 추천 행동(명상/가이드) + 최근 기록
- 녹음: `준비` / `녹음 중` / `일시정지` / `분석 중` / `실패`
- 분석 결과: `요약` + `지표 상세` + `텍스트 조언` + `추천 콘텐츠` + `기록/공유`
- 치유 콘텐츠: `리스트` / `상세` / `플레이어`
- 보관함: `녹음 파일 리스트` / `상세` / `삭제/공유`
- 설정: `계정` / `알림` / `개인정보·데이터(내보내기/삭제)` / `도움말` / `긴급 안내`

## 3) 톤앤매너(Visual + Copy)

### 3.1 비주얼 키워드

- 따뜻한 종이 질감의 뉴트럴(오프화이트) + 자연계열 포인트(티얼/세이지)
- 라운드 코너(12~16) + 얕은 그림자(과한 글로시 금지)
- 차트는 “선명하지만 부드럽게”: 진한 원색 남발 금지, 1~2개 포인트 컬러만 강조

### 3.2 카피 규칙(예시)

- 금지: “정상/비정상”, “위험”, “문제” (의학적 단정/공포 유발)
- 권장: “오늘은 긴장 경향이 조금 높게 나타났어요.” / “최근 7일 평균 대비 +8”
- CTA 문구: “지금 3분 호흡하기”, “짧게 안정 찾기”, “내 기록 보기”
- 결과 화면 하단 고정 안내(권장):
  - “본 결과는 참고용이며 의학적 진단이 아닙니다. 심각한 불안/위기 상황에서는 전문가 도움을 받아주세요.”

##### Spacing(4pt scale)

| Token         | Value(예시)                   | Usage            |

#### 4.1.4 Motion(가이드)

- 마이크로 인터랙션: 150–200ms, 화면 전환: 220–280ms
- easing(권장): `cubic-bezier(0.2, 0, 0, 1)` (부드러운 ease-out)
- 녹음 버튼: press 시 scale 0.96(150ms), release 1.0
- “분석 중” 로딩: 회전/점 3개 등 저자극 애니메이션(과한 바운스 금지)

#### 4.2.1 Mobile (iOS/Android)

- 기본 화면 패딩: 좌/우 `space.4(16)`
- 섹션 간격: `space.6(24)`

- 그리드: 12 columns, gutter 24, 카드 그리드는 2~4열로 반응형 변화
- Breakpoints(권장):
  - `xs 360`, `sm 480`, `md 768`, `lg 1024`, `xl 1280`, `2xl 1440`

- 아이콘 스타일: 라인(Outline) 기반, 모서리 라운딩, 시각적 무게 균일(혼합 금지)

- 선 두께(권장): 1.5~2.0px(웹 기준) / 플랫폼 기본 아이콘 두께에 맞춰 통일

- 그래픽(일러스트/배경): 고채도·자극적 대비 금지, 저채도 그라데이션/추상 형태 권장(명상·안정 맥락 유지)

## 5) 컴포넌트 규격(Components Spec)

### 5.1 Button

#### Variants

- `Primary`: bg `action.primary.bg`, fg `action.primary.fg`
- `Secondary`: bg `bg.surface`, border `border.default`, fg `text.primary`
- `Ghost`: bg transparent, fg `color.brand.600`
- `Destructive`: fg/bg는 `color.danger.*` 사용(필요 화면에만)

#### Sizes

| Size | Height | Padding X | Radius | Font                     |
| ---- | -----: | --------: | -----: | ------------------------ |
| L    |     48 |        16 |     12 | `text.bodyStrong`        |
| M    |     44 |        14 |     12 | `text.bodyStrong`(14~16) |
| S    |     36 |        12 | 999/12 | `text.caption`(12~14)    |

#### States

- Disabled: opacity 0.45 + interaction off
- Loading: 스피너(16) + 텍스트 유지(레이아웃 점프 금지)
- Web hover: 배경 4~6% 어둡게 or border 강조(스케일 변화 금지)

---

### 5.2 Input (Text Field)

- Height: 48, Radius: 12, Border: 1px(`border.default`)
- Label: 상단 `text.caption`, 값과 8px 간격
- Helper/Error: 하단 `text.caption`, 6~8px 간격

상태

- Focus: border `color.brand.500` + focus ring(2px, `focus.ring` 40% opacity)
- Error: border `color.danger.600` + helper `color.danger.600`
- Disabled: bg `color.neutral.10`, text `color.neutral.60`

---

### 5.3 Card

- 기본: bg `bg.surface`, border `border.default`, radius 16, padding 16, elevation 1
- 카드 헤더-본문 간격: 12
- 카드 간 간격(리스트): 12~16

카드 타입(권장)

1. `SummaryCard`(대시보드 요약): 점수 + 추세 + CTA
2. `MetricCard`: 지표명/값/변화(↑↓) + 작은 스파크라인
3. `AdviceCard`: 텍스트 조언 + “짧은 행동” 버튼
4. `ContentCard`: 명상 콘텐츠(썸네일/타이틀/시간)

---

### 5.4 Modal / Bottom Sheet

#### Alert Modal(확인/삭제)

- Width: mobile 320(최대 화면폭-32), web 400~480
- Padding: 20, Radius: 16, Overlay: black 40%
- 버튼: Primary 1개 + Secondary(텍스트) 또는 2버튼(동등 폭)

#### Bottom Sheet(모바일)

- 상단 핸들: 32×4, radius 999
- 상단 radius: 20
- max-height: 90vh, 내부 스크롤 허용
- 닫기: 아래로 스와이프 + X 버튼(접근성)

---

### 5.5 Toast/Snackbar

- 높이: 48, radius 12, padding 12~16
- 위치: 모바일 하단(탭바 위 + safe area), 웹 우하단 또는 상단 중앙(정책 고정)
- 지속: 2.5~4s + 수동 닫기(옵션)

---

### 5.6 Tabs / Navigation

#### Bottom Tab(모바일)

- height 64(+safe)
- icon 24, label 12
- active: `color.brand.600`, inactive: `color.neutral.60`

#### Top App Bar

- height 56, bg surface, divider bottom 1px

---

### 5.7 Waveform(녹음 UI 핵심)

- 높이: 120(기본), 배경: `bg.subtle` 혹은 surface
- 색: 기본 `color.brand.300`, 강조 peak는 `color.brand.600`
- 상태
  - Idle: 잔잔한 고정(혹은 매우 느린 breathing)
  - Recording: 실시간(또는 시뮬) 애니메이션
  - Paused: 정지 + “일시정지” 라벨
  - Processing: waveform blur/딤 + 로딩

---

### 5.8 Charts(결과 시각화)

원칙

- “1개 핵심 점수 + 3~5개 보조 지표”를 1화면에 과밀하게 넣지 않기
- 색은 **브랜드 티얼 1개 + 뉴트럴** 중심. 경고/오류에만 주황/빨강.

권장 차트 구성

1. **Ring Gauge**: 오늘의 마음 안정 지수(0–100)
2. **Line Trend**: 최근 7일 안정 지수 추세
3. **Metric Bars**: 보조 지표(긴장/활력/집중/회복 등) 0–100 막대

표기 규칙

- 축/라벨은 `text.caption`, 값은 `text.bodyStrong`
- 단위/설명(tooltip 또는 info): “0에 가까울수록 ~ / 100에 가까울수록 ~” 형태

---

### 5.9 Audio Player(명상)

- Mini player(옵션): height 64, 하단 고정(탭 위)
- Full player: Bottom sheet 또는 전용 화면
- Controls: 재생/일시정지(44), 10초 전/후(44), 속도(칩), 타이머(칩)
- 배경: 과한 일러스트 금지(저채도 그라데이션/블러)

---

### 5.10 ListItem (기록/보관함)

- 기본 높이: 56 (밀도 높음 리스트는 48, 설정 리스트는 56 권장)
- 좌측: 아이콘(24) 또는 썸네일(40×40, `radius.md`)
- 본문: 제목 1줄(`text.bodyStrong`), 보조 1줄(`text.caption`, `text.secondary`)
- 우측: 상태 배지/시간 + chevron(설정/웹) 또는 더보기(⋯) 액션
- 인터랙션: pressed bg `color.neutral.10`, focus ring 지원(웹), 터치 타겟 44×44 이상
- 스와이프(모바일, 옵션): 삭제/공유 — 기본은 “더보기” 액션을 우선 고려(오작동 방지)

### 5.11 Chip (필터/태그)

- 규격: height 32, paddingX 12, `radius.pill`, font `text.caption`(12~14)
- 기본(Outline): bg `bg.surface`, border `border.default`, text `text.primary`
- 선택(Selected): bg `color.brand.50`, border `color.brand.300`, text `color.brand.700`
- 비활성(Disabled): opacity 0.45, interaction off
- 아이콘 포함: 16 icon + gap 6(전체 높이 유지)

## 6) 화면별 UX/UI 상세(핵심 화면)

### 6.1 온보딩/권한

#### 목표

- “왜 마이크가 필요한지”를 불안 없이 설명 → 동의/거부 모두 존중

#### 화면 규칙

- 한 화면 1메시지 + 1CTA(Primary)
- 권한 거부 시: “설정에서 허용” 안내 + Secondary 버튼 제공

필수 문구(권장)

- “음성은 마음 상태 분석을 위해 사용돼요. 사용자는 언제든 삭제할 수 있어요.”

---

### 6.2 홈/대시보드

#### 레이아웃(모바일)

1. App Bar: 인사(“안녕하세요, ○○님”) + 프로필
2. `SummaryCard`

- 큰 점수(오늘) + “마지막 측정: 2시간 전”
- CTA: “지금 녹음하기”(Primary)

3. `TrendCard`: 7일 라인 차트(미니)
4. `AdviceCard`: 짧은 조언 + “3분 호흡하기”
5. `RecentRecords`: 최근 3개 리스트 + “전체 보기”

#### 레이아웃(웹)

- 2컬럼(≥1024): 왼쪽 요약/조언, 오른쪽 차트/최근기록
- 1컬럼(<1024): 모바일과 동일 흐름

상태

- 첫 사용자(기록 없음): SummaryCard 대신 “첫 기록 만들기” Empty state + CTA
- 로딩: 카드 스켈레톤(높이 고정)

---

### 6.3 음성 녹음 화면

#### 핵심 목표

- 사용자가 “지금 무엇을 하면 되는지” 즉시 이해 (불필요 버튼 제거)

#### 상태별 구성

1. 준비(Idle)

- 안내: “조용한 곳에서 10~20초만 말해보세요.”
- Primary: `녹음 시작`(L)
- 보조: “녹음 팁 보기”(Ghost)

2. 녹음 중(Recording)

- 중앙: 큰 Record 버튼(Stop로 전환) + 파형 + 타이머
- 보조 버튼: `일시정지`(Secondary) (필요 시만)
- 피드백: 시작/정지 시 햅틱(약하게)

3. 일시정지(Paused)

- 파형 정지 + “일시정지됨”
- CTA: `계속 녹음`(Primary) / `종료`(Secondary)

4. 분석 중(Processing)

- 진행: 로딩 + “분석 중이에요(약 10초)”
- 취소는 원칙적으로 숨김(불안/중단 유발) / 필요 시 “취소”는 Ghost로

에러

- 권한 없음: 안내 + “설정으로 이동” 버튼
- 네트워크 오류: “다시 시도” Primary + “임시 저장 후 나중에” Secondary(가능 시)

---

### 6.4 분석 결과 화면

#### 정보 구조(우선순위)

1. 오늘의 점수(요약)
2. 보조 지표(3~5개)
3. 추세(7일)
4. 텍스트 조언(행동 추천)
5. 추천 명상(바로 실행)

#### 레이아웃(모바일)

- 상단: Ring Gauge + 점수(큰 숫자) + 한줄 요약(“오늘은 비교적 안정적인 편이에요”)
- 중단: Metric 카드 2×2 또는 세로 리스트
- 하단: 조언 카드(최대 5줄 + 더보기)
- CTA: “3분 안정 가이드 시작”(Primary)

상태

- 이전 기록 없음: 추세 대신 “기록을 쌓아 추세를 보여드릴게요”
- 부분 실패(지표 일부 없음): 해당 카드 “측정 불가” + info

공유/저장(옵션)

- 공유는 기본 숨김, 사용자가 요청 시: “결과 이미지로 공유”(프라이버시 주의 문구 포함)

---

### 6.5 치유 콘텐츠(명상/가이드)

#### 리스트

- 카테고리(칩): 호흡 / 수면 / 불안 완화 / 집중 / 짧게(3~5분)
- 카드: 썸네일(저채도) + 제목 + 시간 + 난이도
- 정렬: 추천(기본) / 최신 / 짧은 순

#### 상세

- 상단: 제목/설명/길이
- CTA: `재생`(Primary)
- 보조: “다운로드/오프라인”(가능 시)

#### 플레이어

- 진행바(스크러빙 가능), 현재/전체 시간
- 버튼: 재생/일시정지(중앙), 10초 전/후, 속도(칩), 타이머(칩)
- 종료 시: “어땠나요?”(2~3개 이모션 선택은 옵션) → 홈 추천에 반영(향후)

---

### 6.6 보관함(파일/데이터 관리)

#### 리스트 규칙

- 항목 최소 높이 56
- 표시: 날짜/시간, 길이, 결과 요약(작은 배지), 상태(분석 완료/대기)
- 액션: (모바일) 스와이프 삭제/공유 또는 (웹) 체크박스 다중 선택

#### 삭제 UX

- 경고 문구: “삭제하면 복구할 수 없어요.”
- 2버튼 모달: `삭제`(Destructive) / `취소`

필수 항목(권장)

## 부록 B) QA/검수 시나리오 (UX/기능)

> 목적: 권한/네트워크/상태 변화까지 포함하여, 앱/웹에서 동일한 사용자 경험이 보장되는지 확인합니다.

### B.1 온보딩/권한

- 약관/개인정보 동의: 필수 항목 미체크 시 CTA 비활성/에러 안내
- 마이크 권한: 허용/거부/재요청 흐름(설정 이동 포함)
- 권한 거부 상태에서 녹음 진입: 안내 화면 + 대체 CTA 제공

### B.2 녹음

- 시작/일시정지/재개/종료 동작 및 라벨/아이콘 상태 전환
- 백그라운드/전화/알림 등 인터럽트 발생 시 UX(자동 일시정지/안내)
- 권한 없음, 저장공간 부족(가능 시), 마이크 사용 중(다른 앱) 등 예외 처리
- 소음 과다(옵션): 경고 배너/문구 노출, 재녹음 유도(비난 금지)

### B.3 분석(업로드/처리)

- 처리 로딩(스켈레톤/프로그레스) 노출, 예상 소요 안내 문구
- 네트워크 오류: 재시도/나중에 처리(가능 시 임시저장) UX
- 결과 없음/부분 실패(지표 일부 누락): “측정 불가” 상태 + 설명 노출

### B.4 결과(시각화/조언)

- 그래프/차트: 모바일/웹에서 축/라벨/값이 겹치지 않고 가독성 유지
- 텍스트 조언: 길이 변화(짧음/김)에서 레이아웃 안정(더보기/접기)
- 공유/저장(옵션): 민감정보 경고 문구, 공유 범위 안내

### B.5 명상/치유 콘텐츠

- 리스트/필터(Chip)/검색(옵션): 상태 유지, 빈 결과 처리
- 플레이어: 재생/일시정지/seek, 타이머 종료, 속도 변경
- 백그라운드/잠금화면(플랫폼 정책 범위 내)에서 컨트롤/중단 처리

### B.6 파일/데이터 관리(보관함/설정)

- 삭제(단일/다중): 확인 모달, 취소 시 원복, 삭제 후 리스트 갱신
- 공유: 파일/결과 공유 시 프라이버시 문구 표시(옵션)
- 데이터 내보내기/전체 삭제(옵션): 위험 경고 + 2단계 확인(가능 시)

### B.7 접근성(A11y)

- 터치 타겟 44×44 이상(모바일), 웹 포커스 링/키보드 탭 순서 검증
- 스크린리더: 아이콘 버튼/차트 정보 버튼 aria-label(또는 접근성 라벨) 제공
- 색 대비 4.5:1, 텍스트 확대(120~140%)에서 줄바꿈/오버플로우 점검
